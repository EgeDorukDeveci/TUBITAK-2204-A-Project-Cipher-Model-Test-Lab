Main study

Measuring how AI models solve classical ciphers

This project investigates how well current large language models can recover Turkish plaintext from classical cipher systems. The study compares model accuracy across cipher type, text length, key visibility, and model family.

Research question

Can modern AI models solve classical ciphers reliably, and where do they fail when the cipher method, key, or text length changes?

Original study

The original work uses fixed cipher datasets and known model outputs to compare Caesar, Atbash, Vigenere, Playfair, and Autokey results. Hill cipher tests are added as a new extension in this website.

Website role

This website turns the study into a live laboratory. It can run newer models through API calls, save fresh results, compare them with the original baseline, and test harder scenarios where the cipher type and key are hidden.

How it develops the project

By adding repeatable tests, model-family comparisons, exportable results, and multilingual explanations, the website helps the project move from a static report toward a continuously updatable benchmark.

How the website works

From cipher text to benchmark result

A test selects one encrypted dataset row, sends it to the chosen model through the selected API provider, asks the model to recover the plaintext, then sends the answer to GPT-5.4 Nano for evaluation.

  1. Choose a modelPick a Vercel or OpenRouter model, such as ChatGPT 5.5.
  2. Choose a cipher testSelect cipher type, text length, and whether the model sees the key.
  3. Run the API callThe website sends the prompt to the model and records the model output.
  4. Evaluate and compareGPT-5.4 Nano scores the output, then the result appears in New Tests and Comparison.
